Terminal

Consider using a word processor. If you save a file, you might use a keyboard shortcut like cmd/control + s instead of clicking file and then clicking save. While the shortcut doesn't save much time, the true savings is in dealing with multiple files or chaining shortcuts together, e.g. select all, cut, paste.

Terminal saves us time and allows us to interact with our machine by running commands. This is a little different to the Graphical User Interface (GUI) most people are used to.

While it's a little confronting at first, terminal will feel like your second home by the end of the course!
Learning Outcomes

On successful completion of this Unit students will be able to,

  1. understand what the terminal is
  2. use basic commands in terminal

Challenge - Terminal

Using only terminal, navigate your file sytem to:

  1. Create a file structure that matches your family tree e.g. a root folder which is you (use your name). Create two new folders for your parents (use their names). Then add grandparents, etc. See how deep you can go!
  2. Create a folder called students. Inside the folder, create a few student text files 'jane.txt', etc
  3. Rename one of the student files to include full names ('jane doe.txt') - note what happens if you try to use a space. How can you get around this?
  4. Delete one student

Challenge - Running Ruby (IRB)

  1. Open IRB from your command line by running `irb`
  2. Try some basic math (1+1 *enter*)
  3. Quit IRB by sending the `exit` command

Challenge - Running Ruby (From a file)

Let's put our awesome math program into a file we can run any time.

  1. Use terminal to create a new file (e.g. `touch my-cool-app.rb`)
  2. Open the file in your code editor (e.g. `atom my-cool-app.rb`)
  3. Write the same Ruby code (1+1) and save + close the file.
  4. Run the code! We can do this with the `ruby` command. (e.g. `ruby my-cool-app.rb`)
  5. The code runs, but nothing is returned on the screen. Add `puts` to the start of your code and try running the code again! (e.g. `puts 1+1`)
  6. Experiment with some basic ruby!
